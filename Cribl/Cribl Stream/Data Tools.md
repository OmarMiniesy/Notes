### General Notes

These tools create smaller data samples for testing scenarios.
- They are also useful for capturing and sharing datasets across teams.

*Sample Data* can be used to verify if routes and pipelines are properly working.
- **Import Data**: Import sample data by uploading files or data.
- **Edge Data**: Remotely connect to an edge node to pull data from it.
- **Capture Data**: This can be used to capture data as it is coming from its source by creating a relevant filter and then saving it to a file.
	- Capturing can be done at 4 places, at the source, at the destination, or after pre-processing or post-processing pipelines.

*Datagens* is a feature that can be used to generate sample data to troubleshoot.
- Can also create custom *datagens* using a file sample.
- There is also a set of standard *datagens*.

*Simple Preview* shows what happens to data as it passes through a pipeline — displaying both the before and after states side by side to compare how the data looked versus how it will look after processing.
- This is used to test [[Pipelines, Functions, Packs|functions]] and verify their effect before deploying.
